# Vision

JARVIS OS existe pour devenir un assistant IA personnel capable d'automatiser les tâches quotidiennes, d'aider au développement logiciel et de coordonner des agents spécialisés sans perdre la cohérence globale du système.

L'objectif n'est pas seulement de créer un assistant qui répond. L'objectif est de créer un système qui réfléchit avant d'agir, choisit les bons outils, garde une mémoire utile, documente ses décisions et peut grandir pendant plusieurs années.

## Objectif

Créer un système personnel capable de :

- comprendre une demande utilisateur ;
- classifier l'intention ;
- choisir le bon agent ou le bon outil ;
- demander confirmation quand l'action est risquée ;
- exécuter une action quand elle est sûre ;
- apprendre des erreurs et des décisions passées ;
- rester explicable dans ses choix.

## Priorité technique

Ordre de préférence :

1. Local
2. Gratuit
3. Cloud

Les modèles cloud ne doivent être utilisés que lorsque les modèles locaux atteignent leurs limites, ou lorsque la qualité, la sécurité ou la complexité de la tâche le justifie.

## Principes fondateurs

- Le cerveau de JARVIS OS passe avant les workflows.
- Les workflows doivent implémenter une vision, pas remplacer la réflexion.
- Toute décision importante doit être documentée.
- Le système doit être simple au départ, mais extensible dès le début.
- Le routeur intelligent est la pièce centrale.
- Un seul agent au début est acceptable si l'architecture permet d'en ajouter dix ensuite.

## Ce que JARVIS OS n'est pas

JARVIS OS n'est pas un empilement de noeuds n8n, ni une collection de prompts isolés, ni un assistant dépendant d'un seul modèle cloud.

JARVIS OS doit devenir un système orchestré, local-first, documenté et capable de prendre de meilleures décisions avec le temps.
